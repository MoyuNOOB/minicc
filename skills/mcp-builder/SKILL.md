# MCP Builder Skill (Progressive)

## 触发场景
- 用户要构建/改造 MCP server
- 需要把数据库、内部服务或外部 API 以 MCP 能力暴露
- 需要权限控制、可观测与稳定性护栏

## 快速工作流
1) 明确能力边界：资源、工具、权限模型。
2) 设计协议契约：输入/输出 schema、错误模型、超时策略。
3) 最小实现：先跑通一个端到端能力。
4) 补齐护栏：认证授权、审计日志、限流与重试。
5) 验证交付：示例调用、部署说明、回滚策略。

## Progressive Disclosure
### Layer 1（默认）
- 先给最小可运行 MCP 框架与目录建议。
- 只实现核心能力链路，避免一次性过度扩展。

### Layer 2（按需展开）
- 展开资源建模、鉴权策略、错误处理与稳定性策略。
- 输出关键配置项与运行命令。

### Layer 3（深度模式）
- 给生产化方案：多环境配置、监控告警、容量与灾备建议。

## 构建清单
- 契约：输入校验、输出格式、错误码统一。
- 安全：密钥管理、最小权限、审计日志。
- 稳定性：超时、重试、熔断、幂等。
- 可观测：请求日志、指标、trace 关联。
- 可运维：环境配置、部署脚本、回滚路径。

## 输出格式
- 实现方案摘要（范围 + 约束）
- 文件改动建议/代码骨架
- 运行与验证步骤
