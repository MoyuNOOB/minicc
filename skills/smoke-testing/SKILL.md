# Smoke Testing Skill (Progressive)

## 触发场景
- 发布前/部署后快速健康检查
- 用户要求验证核心流程是否可用
- 需要快速发现阻断上线的问题

## 快速工作流
1) 明确核心流程（登录/首页/API关键路径）。
2) 选择最小工具链（Web/API/全栈）。
3) 先写最小用例覆盖核心可用性。
4) 执行并收集失败证据。
5) 输出结论与修复优先级。

## Progressive Disclosure
### Layer 1（默认）
- 只覆盖核心流程的最小冒烟用例。
- 输出是否“可发布/不可发布”的结论。

### Layer 2（按需展开）
- 增补边界校验、失败重试、环境差异配置。
- 提供失败定位建议与重跑命令。

### Layer 3（深度模式）
- 形成稳定的 CI 冒烟策略与门禁规则。

## 测试清单
- 环境可达性：域名/服务/依赖是否在线。
- 核心流程：关键页面、关键 API、关键事务。
- 断言质量：状态码、关键字段、页面关键元素。
- 稳定性：超时、重试、日志与截图证据。
- 发布判定：阻断项与可延期项分离。

## 输出格式
- 执行摘要（通过率 + 阻断项）
- 失败列表（步骤 + 证据 + 初步原因）
- 修复与复测建议
